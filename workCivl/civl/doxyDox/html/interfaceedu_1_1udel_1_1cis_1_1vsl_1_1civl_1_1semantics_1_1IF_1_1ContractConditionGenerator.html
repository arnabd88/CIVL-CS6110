Detailed Description

ContractConditionGenerator provides a set of interfaces for deriving contract clauses.


Terms:

  1. Verifying function: the function which is being verified using contracts.

Abstract interpretation:

All conditions expressed in contracts are evaluate to boolean expression. The evaluation of all kinds of expressions is not guaranteed to have a absolute boolean value (true or false), but they shall have canonical forms so that they can be used to do reasoing and derivation.

Pointer abstract representations:

  1. lambda: Pointer type parameters and global variables will be initialized as an lambda expression: lambda offset : ptr(XP, offset); where XP is an uninterpreted function whose duo inputs are scope ID and variable ID of the parameter or global variable, the output is a unique pointer type expression.
  2. : This class will evaluate expressions to canonical boolean abstract functions.
  3. :This class will evaluate expressions to canonical abstract functions.
  4. lazy initialization: current pointers are initialized at the beginning of verifying a function. Lazy initialization will be considered in the future.

miscellaneous:

  1. Multiple-level pointers: multiple level pointers are pointers point to pointers. Since contract clauses don't have orders, they are parallel conditions. CIVL should be able to figure out dependencies among pointers.

SymbolicExpression edu.udel.cis.vsl.civl.semantics.IF.ContractConditionGenerator.initializePointer ( Variable  pointerVar)

Initialize a pointer type variable v.

v can only be the parameter of the function that is been verifying. v will be initialize as a lambda expression: v = lambda(i) ; lambda(i) := ptr(pointer, i) where the "pointer" is pointer stands for the base address and "i" is the input which stands for an offset. More details can be found at the specification of the class ContractConditionGenerator
