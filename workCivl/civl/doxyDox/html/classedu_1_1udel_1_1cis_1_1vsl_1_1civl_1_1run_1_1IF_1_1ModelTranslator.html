Detailed Description

A model translator parses, links, transforms a list of source files (C or CIVL-C programs) into a ABC program; and then build a CIVL model from that program.

Command line options are also taken into account including macros, transformer settings (e.g., -ompNoSimplify), input variables, system/user include path, etc.

A model translator takes care of a command line section. E.g., the command line civl compare -spec -DN=5 sum.c -impl -DCUDA -inputNB=8 sum.c sum_cuda.c contains two command line sections "spec" and "impl", each of which will then be translated to an ABC program by a specific model translator (different lists of files, different macros, input variables, etc).

Non-compare command line contains one command line section, and thus only one model translator is created.
