Detailed Description

This class translates an AST node of a function body and completes the resulting function accordingly.

The only incomplete translation is the call or spawn statements involved in this function, which dont have the corresponding function of invocation set appropriately.

Author
Manchun Zheng (zmanchun)

Expression edu.udel.cis.vsl.civl.model.common.FunctionTranslator.arrayToPointer ( Expression  array)
inlineprotected

If the given CIVL expression e has array type, this returns the expression &e[0].

Otherwise returns e unchanged.
This method should be called on every LHS expression e when it is used in a place where a RHS expression is called for, except in the following cases: (1) e is the first argument to the SUBSCRIPT operator (i.e., e occurs in the context e[i]), or (2) e is the argument to the "sizeof" operator.
note: argument to & should never have array type.

Parameters
arrayany CIVL expression e
Returns
either the original expression or &e[0]

void edu.udel.cis.vsl.civl.model.common.FunctionTranslator.translateRootFunction ( Scope  systemScope,
ASTNode  rootNode 
)
inline

This method translates the "_CIVL_System" function.

The result should be a function with the following:

  • statements in the global scope, and
  • statements in the main function body.

Initially, the model builder worker have no information about the main function node. Thus the translation starts at translating the rootNode, obtaining a list of initialization statements declared in the root scope and the AST node of the main function.

Parameters
systemScopeThe root scope of the model.
rootNodeThe root node of the AST for translation.
Exceptions
CIVLSyntaxExceptionif no main function node could be found in the rootNode's children.
