Detailed Description

A CIVL transition involves a single atomic statement in one process.

It is to be contrasted with a synchronous transition, which involves two statements executing together in two different processes. It also contains an atomic lock action, which denotes that whether the process is going to grab/release the atomic lock.

Creates a new instance of a CIVL transition.

Precondition: the statement is enabled in the context of the given path condition. There exists a state in the state space such that the process PID with the given identifier has the given atomic, deterministic statement enabled.
